# 🚐 デイサービス送迎 AI自動配車システム【小規模特化・完全無料】

**定員15〜30名程度のデイサービス**向けに作った、**Googleスプレッドシート＋GAS**で動く送迎自動配車ツールです。

毎朝の配車作業を大幅に効率化し、**職員の負担軽減**と**給与アップ**につなげられる現場目線のたたき台です。

**[▶ テンプレートを今すぐコピーする](https://docs.google.com/spreadsheets/d/1bU6F4ZwrLWGPRI6KYABGmCT3DO1KQ9enOAH-NxUkC7E/copy)**

---

### ✨ 導入で期待できる効果

- **配車作業時間を50〜70%削減**（手書き・Excel作業からの解放）
- ベテラン依存（属人化）を解消
- 若手スタッフの残業減少 → **離職防止**と**職場満足度向上**
- 生産性向上により**処遇改善加算の上乗せ要件**を満たしやすくなる
- 小規模事業所（スタッフ6〜8名程度）でも十分運用可能

## ✅ 現在できる主な機能

- 利用者マスタ・契約・送迎条件の一元管理
- 曜日契約から**当日出席者名簿を自動生成**（欠席除外）
- **往路（朝のお迎え）**の自動ルート計算
  - 希望お迎え時間の厳守
  - 帰着期限を考慮したピストン輸送対応
  - 車椅子・助手席の自動割り当て
- **復路（夕方のお送り）**の自動ルート計算
- NG乗客・NGドライバーの双方向回避（スペース除去処理付き）
- 住所から**緯度経度を自動取得**
- 運行実績の自動蓄積

---

## ❌ 現状の制限（改善希望ポイント）

- 距離計算が直線距離（Google Maps Distance Matrix API対応で大幅向上可能）
- NG乗客は現在1名のみ指定可能
- ドライバーと車両の自動紐付けは未実装
- 当日キャンセル後の自動再計算は未対応

## 📋 シート構成

| シート名                    | 役割 |
|---------------------------|------|
| `m_user_basics`           | 利用者基本情報（住所・緯度経度） |
| `t_user_contracts`        | 曜日ごとの利用契約 |
| `t_user_pickup_conditions`| 送迎条件（時間・車椅子・NG設定） |
| `m_vehicles`              | 車両マスタ（定員・時間設定） |
| `m_drivers`               | ドライバーマスタ |
| `t_daily_attendance`      | 当日の出欠管理 |
| `t_routing_outputs`       | 配車結果出力 |
| `t_past_records`          | 運行実績蓄積 |

## 🚀 使い方（5分で開始可能）

1. 上記のテンプレートをコピー
2. 拡張機能 → Apps Scriptを開き、`main.gs`の内容を貼り付け
3. `t_daily_attendance`シートのH1〜H3に施設情報（日付・緯度経度）を入力
4. **「名簿作成」**を実行 → 当日利用者を自動展開
5. **「往路計算」** → **「復路計算」**を実行
6. 必要に応じて微調整後、**「実績保存」**を実行

詳細は後半の「使い方」セクションを参照してください。

---

## 📐 ルート計算のロジック

- **時間指定優先**（希望時間終了が早い人を優先）
- **近傍探索**（時間指定がない人は現在地から最寄りを選択）
- 車両容量・車椅子対応・NG制約をすべて考慮
