# 🚐 デイサービス専用 AI自動配車システム

Google スプレッドシート ＋ Google Apps Script（GAS）で動く、デイサービスの送迎ルートを自動計算するオープンソースのたたき台です。

---

## ⚠️ このシステムについて

週2日、介護タクシー事業のデイサービスで働きながら「毎朝ベテランが頭の中だけで配車を組んでいる」現場を見て作りました。

コードを書きながら「なぜ属人化するのか」が体感できました。条件が多すぎるんです。完璧なシステムを目指すと永遠に完成しない。なのでたたき台として公開します。

デイサービスの現場に関わっている方、ITが得意な方、ぜひ改善してもらえると嬉しいです。

---

## ✅ 現在できること

- 利用者マスタ・契約・送迎条件の一元管理
- 曜日契約から当日出席者名簿を自動生成（欠席者を除外）
- 往路（朝のお迎え）の自動ルート計算
  - 希望お迎え時間（開始・終了）の厳守
  - 帰着期限（次の便の出発時間）を考慮したルート制御
  - ピストン輸送（複数便）の管理
  - 車椅子利用者の対応車両への自動割り当て（♿ / 💺 表示）
  - NG乗客・NGドライバーの回避（双方向・完全一致チェック）
- 復路（夕方のお送り）の自動ルート計算
  - 送り届け期限（dropoff_time_end）を考慮
  - NG制限も往路と同様に対応
- 住所から緯度経度を自動取得（GAS組み込みジオコーダー使用・無料）
- 運行実績の自動蓄積

---

## ❌ 現状できないこと（改善してほしい点）

- **距離計算が直線距離**のため、山間部・島嶼部では精度が落ちる（Google Maps Distance Matrix API対応で改善可能）
- **NGペアは1人のみ指定可能**（複数指定には未対応）
- **ドライバーと車両の当日紐付け**（現状は手動運用前提）
- **キャンセルの当日入力後に自動再計算されない**（手動で再実行が必要）
- **狭い道・駐車スペース・玄関の状況**などの地理的暗黙知は反映されない
- **利用者・ドライバーの相性**など人間関係の暗黙知は反映されない

## 📋 シート構成

| シート名 | 役割 |
|---|---|
| `m_user_basics` | 利用者基本マスタ（住所・緯度経度など） |
| `t_user_contracts` | 曜日別の利用契約 |
| `t_user_pickup_conditions` | 希望時間・車椅子区分・NG設定など |
| `m_vehicles` | 車両マスタ（定員・出発時間・帰着期限） |
| `m_drivers` | ドライバーマスタ（※現在AI配車では未使用） |
| `t_daily_attendance` | 当日の出欠テーブル |
| `t_routing_outputs` | 配車計算結果の出力先 |
| `t_past_records` | 運行実績の蓄積 |

## 🚀 使い方

### 1. スプレッドシートをコピーする

[▶ スプレッドシートのコピーを作成](https://docs.google.com/spreadsheets/d/1w3OPRpoBGJYzsFHRjuFxIsxo9TPaYZzpr9Z9lPLM81Y/copy)

※ テンプレートには架空の利用者データが入っています。実際に使う際は全て差し替えてください。

### 2. GASスクリプトを貼り付ける

1. スプレッドシートのメニューから「拡張機能」→「Apps Script」を開く
2. このリポジトリの `main.gs` の内容をコピペして保存

### 3. 施設情報を設定する

`t_daily_attendance` シートの右上（H列）に以下を入力します。

| セル | 内容 | 例 |
|---|---|---|
| H1 | 運行日 | 2026/06/01 |
| H2 | 施設の緯度 | 34.844 |
| H3 | 施設の経度 | 135.503 |
| H4 | 施設の住所（表示用） | 大阪府箕面市〇〇 |

### 4. 毎日の手順

1. H1セルの日付を今日に変更
2. 「名簿作成」を実行 → 曜日契約から自動で出席者を展開
3. 欠席者の `attendance_status` を「欠席」に変更
4. 「往路計算」を実行 → ルートが出力シートに表示される
5. 内容を確認して必要なら手動で微調整
6. 「復路計算」を実行
7. 「実績保存」を実行 → 運行結果を蓄積

## 📐 ルート計算のロジック

### 往路（朝のお迎え）

**Mode A（時間制限あり）**：希望お迎え時間の終了が早い人から順番に優先して乗せます。帰着期限に間に合わない場合はその便を切り上げて次の便へ移行します。

**Mode B（時間制限なし）**：時間指定のない人は、現在地から最も近い人から順番に乗せます（近傍探索）。

### 復路（夕方のお送り）

往路と同じMode A/B構造で、送り届け期限（dropoff_time_end）が早い人を優先します。

### 距離計算

現状は直線距離（緯度経度から近似計算）を使用しています。都市部・住宅街では実用上の誤差は小さいですが、山間部や島嶼部では Google Maps Distance Matrix API の導入を推奨します。

---

## 🔧 カスタマイズのヒント

**Google Maps APIを使いたい場合**

`getDistance()` 関数を Distance Matrix API の呼び出しに差し替えることで、実際の道路距離での計算が可能になります。APIキーとカード登録が必要ですが、25名規模では無料枠内で収まります。

**ドライバーと車両を紐付けたい場合**

`m_vehicles` にdriver_idカラムを追加し、`m_drivers` と照合するロジックを `runAiRouting()` に追加してください。

**通知機能を追加したい場合**

GASのMailApp（メール）やLINE Messaging API（Make経由）を組み合わせることで、ドライバーへのルート通知を自動化できます。
